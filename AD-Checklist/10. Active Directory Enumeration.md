# Active Directory Enumeration

## Objective

Enumerate the Active Directory environment to identify attack paths, misconfigurations, excessive privileges, trust relationships, delegation settings, and privilege escalation opportunities.

---

# 1. BloodHound Enumeration

Collect Active Directory objects and relationships.

## Linux (BloodHound.py)

```bash
bloodhound-python -u <USER> -p <PASSWORD> -d <DOMAIN> -dc <DC_HOST> -c All -ns <DC_IP>
```

## Windows (SharpHound)

```powershell
.\SharpHound.exe -c All
```

or

```powershell
Invoke-BloodHound -CollectionMethod All
```

> **Tip:** Re-run BloodHound after every privilege escalation or lateral movement. Newly compromised accounts often reveal additional attack paths.

---

# 2. Enumerate Domain Information

Gather basic domain information.

```bash
nxc ldap <DC_IP> -u <USER> -p <PASSWORD> --groups
```

```bash
nxc ldap <DC_IP> -u <USER> -p <PASSWORD> --users
```

```bash
nxc ldap <DC_IP> -u <USER> -p <PASSWORD> --computers
```

```bash
nxc ldap <DC_IP> -u <USER> -p <PASSWORD> --dc-list
```

Identify:

- Domain Controllers
- Domain SID
- Domain Functional Level
- Domain Users
- Domain Computers
- Domain Groups

---

# 3. Group Enumeration

Review privileged and interesting groups.

Focus on:

- Domain Admins
- Enterprise Admins
- Schema Admins
- Account Operators
- Server Operators
- Backup Operators
- Print Operators
- Remote Management Users
- Remote Desktop Users
- DNSAdmins

Identify:

- Nested group membership
- Service accounts
- Unexpected privileged users

---

# 4. ACL Enumeration

Look for permissions that can be abused.

Interesting permissions include:

- GenericAll
- GenericWrite
- WriteOwner
- WriteDACL
- ForceChangePassword
- AddMember
- AddSelf
- AllExtendedRights

BloodHound will highlight these relationships automatically.

---

# 5. GPO Enumeration

Enumerate Group Policy Objects.

Look for:

- Startup scripts
- Logon scripts
- Scheduled tasks
- Registry Preferences
- Drive mappings
- Software deployment
- Local administrator configuration

Inspect:

- SYSVOL
- NETLOGON

Check for:

- Group Policy Preferences (GPP)
- `cpassword`
- Writable GPOs

---

# 6. Trust Enumeration

Identify domain and forest trust relationships.

Review:

- Parent/Child trusts
- External trusts
- Forest trusts
- Shortcut trusts

Determine:

- Direction
- Transitivity
- SID filtering

Trusts can provide attack paths into additional domains.

---

# 7. Delegation Enumeration

Identify systems and accounts configured for Kerberos delegation.

Review:

- Unconstrained Delegation
- Constrained Delegation
- Resource-Based Constrained Delegation (RBCD)

Look for:

- TrustedToAuthForDelegation
- AllowedToDelegateTo
- AllowedToActOnBehalfOfOtherIdentity

---

# 8. Service Principal Names (SPNs)

Enumerate service accounts.

```bash
nxc ldap <DC_IP> -u <USER> -p <PASSWORD> --kerberoasting kerberoast.txt
```

Look for:

- MSSQL
- HTTP
- CIFS
- HOST
- Exchange
- Custom services

Service accounts frequently have elevated privileges.

---

# 9. AS-REP Roastable Users

Identify users with Kerberos pre-authentication disabled.

```bash
GetNPUsers.py <DOMAIN>/ -dc-ip <DC_IP> -usersfile users.txt -request
```

---

# 10. Certificate Services (AD CS)

Determine whether AD CS is deployed.

```bash
certipy find -u <USER>@<DOMAIN> -p <PASSWORD> -dc-ip <DC_IP>
```

Look for:

- Vulnerable templates
- Enrollment rights
- ESC1–ESC16 misconfigurations

---

# 11. Machine Account Quota

Determine whether standard users can create computer objects.

Review:

- `ms-DS-MachineAccountQuota`

A non-zero value may enable attacks such as RBCD.

---

# 12. LAPS Enumeration

Determine whether Local Administrator Password Solution (LAPS) is deployed.

Check:

- Legacy LAPS
- Windows LAPS

Identify users with permission to read LAPS passwords.

---

# 13. DNS Enumeration

Review:

- DNS zones
- Host records
- Hidden servers
- Management interfaces

DNS often reveals additional attack surfaces.

---

# 14. Session Enumeration

Identify logged-in users.

```bash
nxc smb <TARGET_IP> -u <USER> -p <PASSWORD> --sessions
```

Look for:

- Domain Admin sessions
- Service accounts
- Privileged users

---

# 15. Local Administrator Enumeration

Determine local administrator relationships.

```bash
nxc smb <TARGET_IP> -u <USER> -p <PASSWORD> --local-groups
```

Identify:

- Local Administrators
- Remote Desktop Users
- Remote Management Users

---

# 16. Attack Path Analysis

After collecting all data, review BloodHound for:

- Shortest Paths to Domain Admin
- Owned Principals
- Kerberoastable Users
- AS-REP Roastable Users
- High Value Targets
- RBCD Opportunities
- Shadow Credentials
- AD CS
- DCSync Rights
- ForceChangePassword
- GenericAll
- GenericWrite

Always investigate the **shortest and lowest-noise** attack path first.

---

## Tips

- Re-run BloodHound after every privilege escalation.
- Enumerate before exploiting.
- Review every ACL identified by BloodHound.
- Never ignore GPOs or SYSVOL.
- Always inspect trust relationships.
- Check delegation before attempting Kerberos attacks.
- Review logged-in sessions regularly.
- Look for multiple attack paths—there is rarely only one.
